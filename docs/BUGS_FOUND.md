## Bug Priority Definitions

### Critical (P0)
- Application crashes or is unusable
- Data loss or corruption
- Security vulnerabilities
- Cannot complete core user flows
- **Action:** Fix immediately, block release

### High (P1)
- Major features broken
- Significant user experience issues
- Workarounds difficult or unreliable
- Affects many users
- **Action:** Fix before release

### Medium (P2)
- Minor features broken
- Cosmetic issues that affect usability
- Workarounds available
- Affects some users
- **Action:** Fix if time permits, or in next release

### Low (P3)
- Minor cosmetic issues
- Enhancement requests
- Rare edge cases
- Affects few users
- **Action:** Nice to have, backlog
